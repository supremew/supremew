2021-12-23 archlinux 升级后，启动 lightdm 失败

Posted on 2021-12-23 Edited on 2022-01-12 In 日常生活

先来说说， 如何出现 lightdm 启动失败的问题的过程。

  1. 在发现 qutebrowser 浏览器有新版本后， 于是单独升级了 qutebrowser.
  2. 升级 qutebrowser 后， 发现 qutebrowser 不能用了， 估计是新版本的 qutebrowser 使用了新版本的某一个 python 包， 为了偷懒解决该问题。
  3. 于是对系统进行了全面的升级， sudo pacman -Syu
  4. 在重启后，发现图形界面不出来了， 一查启动过程中的错误。 sudo systemctl jounalctl -b -p err
  5. 才发现 lightdm 启动失败了。

一般情况下， lightdm 启动不了是显卡驱动的问题， 于是查了下 nvidia 模块 lsmod |grep nvidia，果然没有 nvidia 模块， 但是通过查看 nvidia 安装情况 pacman -Q|grep nvidia，是安装了。 于是上 archlinux 的 wiki 查了一下关于 nvidia 的资料， 发现 nvidia 驱动在新版本中不再支持 780 的显卡了， 和 780 显卡需要安装另外一个安装包。 nvidia-77xx-dkms

按照说明安装了 nvidia-470xx-dkms yaourt -S nvidia-470xx-dkms, 最终得以解决。

2021-12-13 git 的默认编辑器的修改

Posted on 2021-12-13 Edited on 2021-12-23 In 日常生活

git commit

此时是进入GUN nano编辑器。在这里可以添加你的commit imformation 然后ctrl+o,回车保存，再ctrl+x退出。 因为对Nano编辑器很不熟悉，在这里我想将默认编辑器改为vim。打开.git/config文件，在core中添加 editor=vim即可。
或者运行命令 git config –global core.editor vim 修改更加方便。

git commit -a

把所有add了的文件都加入commit，然后进入编辑器编辑commit信息。

git commit -m “commit imformation”

直接在后面添加commit 信息然后提交commit，与gitcommit相比快捷方便，但是就是commit信息格式无法控制。

git commit –amend

修改最后一次commit的信息。

2021-11-30 arch 升级失败后修复过程

在 2021-11-29 因升级系统过程中出现电脑卡死，不得不重新启动，　最终导致升级失败，　重启后无法正常进入程序。　主要问题体现问 /boot 目录下没有相应的内核以及启动镜像的存在。

下面为解决办法

  1. 准备启动镜像，使用 ventoy 并制作启动优盘

  2. 启动并进入 arch 的 live 系统。

  3. 连接网络，　这里我所使用的是 wifi

  4. 挂载根分区到 /mnt 目录下。

  5. 挂载其他分区

  1. 重新升级
  1. 双重确认是否存在有损坏的包。
  1. 重新安装有损坏的包。
