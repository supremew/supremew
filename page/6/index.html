importlib.find_loader(name, path=None) Find the loader for a module, optionally within the specified path. If the module is in sys.modules, then sys.modules[name].loader is returned (unless the loader would be None or is not set, in which case ValueError is raised). Otherwise a search using sys.meta_path is done. None is returned if no loader is found. A dotted name does not have its parents implicitly imported as that requires loading them and that may not be desired. To properly import a submodule you will need to import all parent packages of the submodule and use the correct argument to path. New in version 3.3. 2020-06-26 archlinux 中文问题

Posted on 2020-06-23 Edited on 2021-12-11 In 日常生活

首先说， 在 archlinux 的 wiki 中已经说的很清楚了， 但是在实际操作中还是会遇到中文的问题 archlinux 中安装了 deepin deepin - extra 在终端中粘贴入中文， 不显示中文， 在vim 中粘贴入中文显示问号， 检查终端设置是正常的， vim 设置也是正常的， 看了 wiki 后， 觉着是 locale 的问题， 现在就说一哈解决办法。 再终端中执行 locale ， 看是否是与模拟终端中执行的结果相同。

  1. 我在 /etc/locale.conf 中设置成

LANG=en_us. UTF-8

  1. 在终端中执行 locale，基本输出结果都符合预期。
  2. 在deepin 的虚拟终端中执行 locale，输出结果与预期不一致。
  3. 分析， 在进入 deepin 之前 设置对的， 而在 进入deepin 之后是错的， 因此估计是 用户文件的问题。(/home/用户名)
  4. 先删除用户及用户文件， 重新建立用户及用户文件。

userdel -r archlinux user add -m archlinux

2019-12-25 archlinux 的安装

Posted on 2019-12-25 Edited on 2021-12-11 In 日常生活

总的来说， archlinux 的安装还是比较顺利的， 现在来说说在安装过程遇到的几个问题及解决办法。

  1. 具体的安装过程得参考 archlinux wiki
  2. 在启动U盘的制作过程， 遇到了的问题是 U 盘制作了， 但是在引导启动时老是出问题， 卡住了。 键盘两个灯同时闪烁。出现这样的问题， 果断换一台机子再重新制作。（一定要严重下载镜像的完整性， 出现该问题的原因我想是镜像文件不完整。）
  3. 我所安装 archlinux 的机子为台式机， 外接 usb 的无线网卡， 其芯片 rtl8812au， 在安装无线网卡的过程中遇到了不小的麻烦， 在 github 找了很多该芯片的驱动都没有搞好。 搞了一晚上后， 灵光闪现， 意识到不同的linux内核可能需要不同的驱动。 因此在github找到了一个作者， 该作者写了好几个版本的驱动， 有 4.3.4 , 5.6.4.1, 等等。 于是我将其都下载下来， 一个一个的试。 找到了适合我网卡的驱动。
