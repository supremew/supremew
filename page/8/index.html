2018-09-09 qutebrowser 的编译与安装

Posted on 2018-09-09 Edited on 2021-12-11 In 日常生活

Qutebrowser 完全不像其他这些浏览器。如果你遇到过 Vim 这样的轻量级浏览器，它基本上就是这样子――简单的界面、简约的设计，而且只要使用键盘就能够处理任何事务。 尽管 qutebrowser 的作者推荐我们使用 qt 5.11.1， pyqt 5.10.1， 但是通过相当长的一段时间的使用后， 还是推荐大家使用 qt 5.10.1 , pyqt5 5.10.1。 （推荐使用 qt 5.10.1， pyqt5 5.10.1 的主要原因是 adobe 公司已经宣布将在 2020 年停止对 adobe flash player 的支持， adobe 即将退出历史舞台。 然后现在我们的大部分视频网站还在使用 adobe flash player 技术， 因此如果使用 qt 5.11 以后的版本将会遇到一些小问题， 比如视频不能全屏播放。） 由于不同 linux 版本的更新进度不同， 因此所使用的 qt 版本也有高低不同， 因此需要源代码编译。 下面就是一个简单的编译过程。（因为 qutebrowser 的作者写的代码是遵守开源标准的， 因此要源代码编译时相当的简单。） 其步骤为：

  1. 到 qt 官网下载最新版的 qt 5.10, 并安装。如下载下来的文件为： qt-opensource-linux-x64-5.10.1.run
    • chmod +x qt-opensource-linux-x64-5.11.1.run
    • ./qt-opensource-linux-x64-5.11.1.run
  2. 安装 qtchooser （qtchooser 是用于管理多版本的 qt 的工具包。）。
    • sudo apt-get install qtchooser
    • sudo qtchooser -install 5.10.1 /安装qt版本的qmake路径/qmake
    • cd /usr/lib/x86_64/qt-chooser/
    • sudo rm -rf default.conf
    • ln -s ../qt-default/qtchooser/5.10.1.conf default.conf
  3. 到 PyQt 官网下载 PyQt5 5.10.1， 下载 sip 4.9.12 ， 并按照下面的步骤进行编译安装。
    • 首先安装 sip 4.9.12
      • cd sip-4.9.12
      • python3 configure.py
      • make
      • sudo make install
    • 再安装 PyQt5 5.10.1
      • cd PyQt5-5.10.1
      • python3 configure.py
      • make
      • sudo make install
  4. 到 github.com 中去 clone qutebrowser 的源代码， 并进行安装
    • cd qutebrowser
    • python3 script/asciidoc2html.py
    • python3 setup.py build
    • python3 setup.py install –record install_files.txt –user
  5. 再编译安装 PyQt5 中遇到的问题及解决办法
    1. 在启动 qutebrowser 时， 出现 SystemError: intialization of QtCore 的问题。 这是因为 PyQt5 编译不当造成的。注意在编译 5.11 及以后的版本需要用 python3 configure.py –sip-moduler PyQt5.sip 。然后再重新编译 PyQt5 。
    2. 出现 sip 的版本问题的问题时， 注意检查一下系统是否自带了 python-sip， 或者 python3-sip ， 或者 python-sip-dev， python3-sip-dev 等软件包， 直接将他们卸载就行了， 最好用 sudo apt-get purge python-sip-dev pthon3-sip-dev python3-sip python-sip

2018-09-06 关于 deepin linux 的看法

Posted on 2018-09-06 Edited on 2021-12-11 In 日常生活

前段时间适用了一下 deepin linux 15.7 发现该 linux 还是可以， 桌面制作的非常的华丽， 感觉上还是相当的不错。 deepin linux 根 ubuntu 一样，均是由 debian 再次开发而来。 不过 deepin linux 跟那些有大社区支持的 linux 版本不能比， deepin linux 的开发社区还不强大。 这体现在如下方面： * 更新源的版本不是最新 * 有时候必须依赖的包版本过低。(好像是一个 lib6c-dev 的安装包的依赖问题， 该问题可以通过重新安装一下该包就可以了， 如sudo apt-get install --reinstall lib6c-dev) 下面我讲在安装 deepin linux ， 和使用他时遇到的问题进行记录一下。

  1. 像我这样， 经常在英文环境下使用， 但是又需要输入中文的， 你就需要这样进行安装. 首先安装 deepin linux 时， 先在中文环境下安装完成， 待安装好了之后再讲环境（languge and keyboard）设置成英文的就好了。 否则会遇见在英文环境下怎么也不能用fcitx进行中文输入， 必安装下才能进行输入。
  2. 在安装时 deepin 时， 需要先讲已经自动挂载的分区进行卸载， 要不然在安装时是看不到相应的分区的。
  3. 在有些程序中时不能用 fcitx 进行中文输入的， 这时候需要将 /usr/lib/x86_64/qt5/plugins/platforminputcontent/中的libfcitx_.so 替换掉， 用 ubuntu xienl 中的 libfcitx_.so 这主要是因为版本的问题。
  4. 关于 sogou 黑色边框的问题。 出现这样的问题， 一般是因为安装闭源驱动未完全而导致， 比如我是 nivida 的显卡， sudo apt-get install nivida-driver, 重启， 在用 deepin 自带的显卡驱动管理器进行切换， 到闭源， 就能解决的。（笔记本的双显卡需要选择 prime 版本）
  5. 由于手贱， 将efi分区下的efi/ubuntu/下的所有文件都删除了（我有两块硬盘， sda， sdb， 这两块硬盘的第一个盘都是搞得efi分区， 结果我都将其下面的efi/ubuntu给删除了， 导致怎么也启动不了。）， 导致了grub2不能加载引导配置文件grub.cfg而不能成功引导系统启动。 其解决办法是到相应的分区下， 创建ubuntu目录， 并将deepin目录下的grub.cfg复制到ubuntu目录下， 就能成功引导了。
  6. 在启动过程中出现 uefi can’t get size 的问题是因为没有开启bios的安全启动模式。
  7. 出现 can’t open access to console , the root account is locked ， 这里我写了详细的解决办法。
  8. 在使用sougou拼音时， 容易出现卡死的现象， 这里也同样有解决办法 我在这个帖子里面提供了解决办法
  9. 出现需要升级 intel-microcode 的提示， 可以直接采用 sudo apt-get install intel-microcode

2018-02-21 uefi模式下安装ubuntu17.10 详细记录

Posted on 2018-02-21 Edited on 2021-12-11 In 日常生活

目前越来越多的笔记本， 或者是台式机都支持uefi模式， 甚至很多已经默认在uefi模式下工作， 有的已经不在兼容传统模式了。现我就将我在新买的联想本本上折腾安装ubuntu17.10的过程记录下来， 希望能够帮助后来人。（网上很多教程， 但是大部分的教程都是将你往坑里面带的， 但是我希望这教程不会将你带入坑！）

  1. 配置优盘， 准备一个至少4g的优盘。 采用gdisk或者fdisk分区工具将u盘的分区表改成gpt格式， 并创建两个分区（u盘总计两个分区）， 第一个分区格式为fat32， 第二个分区为fat32（这个分区也可以是ntfs格式的）——-注意分区表格式一定得是gpt， 第一个分区的文件系统格式必须是vfat也就是fat32。 我采用的分区工具是fdisk， 现我将具体命令列出， 供参考。
  1. 下载ubuntu 17.10， 这里我下到的官方版， ubuntu-17.10.1-desktop-amd64.iso， 现我将其放置于我的u盘的第二个分区里面的根目录下面。用解压工具winrar或者ultraiso等解压该iso文件并将其中的boot/ 和 efi/ 文件夹复制到u盘的第一个目录去； 然后将ubuntu-17.10.1-desktop-amd64.iso复制到u盘的第二个分区根目录下。 （说明下： 由于历史原因， 在windowns下面是看不到u盘的第二个分区， 这个时候你可以用分区工具gdisk工具将其复制到第二个分区， 具体操作我就不多说了。 在ubuntu下的话， 可以用mount来挂载u 盘的分区到指定文件夹下面.）
  2. 重启系统, 我的本本是按F2, 进入bios， 然后禁用security boot， 再按 f10 保存退出， 然后狂按f12。选择u盘启动， 进行grub2. 然后按c, 进入grub2命令行模式， 采用命令引导。
  1. 等后ubuntu加载, 后面就是一路next， 不过在分区时， 一定要注意不要将你的windows的分区以及其备分分区覆盖。一定要创建一个efi系统分区， 否则会启动不了的； 还择启动器时要选择你的磁盘，我的是/dev/sda, sda后面还能有数字。（分区说明， 现在已经没有必要像以前那样分很多分区了， 那已经成为了过去。 我就分了3个分区， / 根分区， /home 分区， efi类型的分区。）
